CSOPESY · Desktop OS Emulator

System Architecture

Figure 1 — Layered Architecture

A four-layer design. Each upper layer depends only on the layer beneath it: Core wraps the libraries, Compositor directs the render order, Shell builds the desktop chrome, and Apps are the windowed programs.

Figure 2 — Per-Frame Render Pipeline

The emulator is a single-threaded, immediate-mode loop running at ~60 fps. Everything is redrawn from scratch every frame — there is no retained widget tree. The boot gate early-returns until the FSM finishes, so the desktop never renders a partial frame.
